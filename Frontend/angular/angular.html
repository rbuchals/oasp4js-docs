What is a Module?

You can think of a module as a container for the different parts of your app – controllers, services, filters, directives, etc.

How to deal with modules?

Module declaration


angular.module('myFirst', ['dep1','dep2','dep3']);

Module retrieval


angular.module('myFirst');

Application bootstrap

Config vs Run


angular.module('myModule', []).
    config(function(injectables) {
      // This is an example of config block.
      // You can have as many of these as you want.
      // You can only inject Providers (not instances)
      // into config blocks.
    }).
    run(function(injectables) {
      // This is an example of a run block.
      // You can have as many of these as you want.
      // You can only inject instances (not Providers)
      // into run blocks
    });

Recommended Setup

  • A module for each feature
  • A module for each reusable component (especially directives and filters)
  • And an application level module which depends on the above modules and contains any initialization code

Dependency Injection

Dependency injection is a software design pattern that implements inversion of control and allows a program design to follow the dependency inversion principle

Factory Recipe #Object


angular.module('myFirst', [])
    .controller('MyFirstCntl', function ($scope, userService) {
        $scope.fullName = userService.getFullName();
    })

            surname: 'Nowak'

                return user.name + ' ' + user.surname;
            }
        }
    });

Service vs Factory


function UnicornLauncher(apiToken) {
    this.launchedCount = 0;
    this.launch = function () {
        //uses apiToken somehow
        this.launchedCount++;
    }
}

...instantiate as Factory


angular.module('myFirst', [])
    .factory('unicornLauncher', ["apiToken", function(apiToken) {
        return new UnicornLauncher(apiToken);
    }]);

...instantiate as Service


angular.module('myFirst', [])
    .service('unicornLauncher', ["apiToken", UnicornLauncher]);

Provider Recipe

Lets say we want to preconfigure our unicorn launcher...


angular.module('myFirst', [])
    .provider('unicornLauncher', function () {
        var galaxyName = 'the Milky Way';

        this.setGalaxyName = function (galaxy) {
            galaxyName = galaxy;
        };

        this.$get = ["apiToken", function (apiToken) {
            return new UnicornLauncher(apiToken, galaxyName);
        }];
    });
    

angular.module('myFirst')
    .config(function (unicornLauncherProvider) {
        unicornLauncherProvider.setGalaxyName('Andromeda');
    });


angular.module('myFirst')
    .controller('DemoCntl', function ($scope, unicornLauncher) {
        unicornLauncher.launch();
    });

Angular Routes

Angular Routes - Simple routing


angular.module('myFirst', []).config(function ($routeProvider) {
    $routeProvider.when('/tables', {
        templateUrl: '/table-mgmt/html/tables.html',
        controller: 'TablesCntl'
    }).when('/table-view', {
        templateUrl: '/table-mgmt/html/table-details.html',
        controller: 'TableDetailsCntl'
    });
});

Angular Routes - Initial data


angular.module('myFirst', []).config(function ($routeProvider) {
    $routeProvider.when('/books', {
        templateUrl: '/table-mgmt/html/tables.html',
        controller: 'TablesCntl',
        resolve: {
            tables: function(){
                //return data or promise and inject into controller
            }
        }
    })
    .controller('TableDetailsCntl', function(tables){
        //for instance
        $scope.tables=tables;
    });
});

        //use $routeParams.tableId

Angular Backend Communication - $http


angular.module('myFirst', [])
    .factory('tables', function($http){
        return {
            getTables: function () {
                return $http.get('http://yourserver.com/tables')
                    .then(function (response) {
                        return response.data;
                    });
            }
        };
    });

$http provides ...

...generic method $http(config)
... and shortcut methods

  • $http.get(url, [config])
  • $http.head(url, [config])
  • $http.post(url, data, [config])
  • $http.put(url, data, [config])
  • $http.delete(url, [config])
  • $http.jsonp(url, [config])
  • $http.patch(url, data, [config])

$httpBackendProvider allows...

... to set default headers for every type of request

... to define interceptors for cross cutting feature

... to enable XSRF protection
